# 付款与协议

## 概述

Basalt OS 的付款体系分为两大类：**协议代扣**（体系内子户间划转，免费）和**授权代付**（向体系外付款，0.05% 手续费）。两者在签约方式、资金流向、费率、状态体系上存在本质差异。

协议代扣用于体系内资金归集和服务费上缴；授权代付用于门店向外部银行卡/支付宝付款（如租金、工资、分红等）。两者都需要先完成合约签约，才能发起付款操作。

---

## 操作步骤

### 一、协议代扣（余额划转）

#### 业务场景

- 服务费上缴（门店 → 总部）
- 采购调拨（门店 → 区域公司）
- 其他体系内子户划转

#### 签约流程（商户主动发起）

```
商户在网商云资金小程序发起签约
    ↓
网商通过授权/解约通知接口(ant.mybank.merchantprod.merchant.arrangement.info.notify)通知平台
    (ArrangementStatus=VALID)
    ↓
系统创建签约申请单并记录合约号 ArrangementNo
    ↓
合约状态为"已生效"
```

- 签约发起方：商户（在小程序主动签约）
- 平台侧无主动签约接口，只能被动接收通知
- 收到通知后应立即调用协议代扣授权状态查询接口（`ant.mybank.merchantprod.merchant.arrangement.info.query`）再确认一次合约最新状态

#### 解约流程（需平台审核）

```
商户在小程序发起解约
    ↓
网商通知平台（ArrangementStatus=INVALID_TO_BE_CONFIRM）
    ↓
系统创建解约申请单，合约状态变为"解约中待平台确认"
    ↓
平台调用解约申请审核接口(ant.mybank.merchantprod.merchant.arrangement.audit)进行审核
    ↓
同意（AGREE）→ 合约失效（"已失效"）
拒绝（REJECT）→ 合约恢复"已生效"
```

#### 代扣付款流程

```
调用协议授权代扣接口(ant.mybank.bkcloudfunds.order.withhold.apply)
    → 状态"处理中"
    → 通过代扣通知结果接口(ant.mybank.bkcloudfunds.protocol.withhold.result.notify)接收结果
       或调用代扣查询结果接口(ant.mybank.bkcloudfunds.protocol.withhold.query)主动查询
    → "成功" 或 "失败"
```

**协议授权代扣接口核心入参**：

| 参数 | 说明 | 示例 |
|------|------|------|
| Scene | 业务场景 | PROTOCOL_WITHHOLD |
| TotalAmount | 交易金额（单位：分） | - |
| PayerId | 付款方商户号 | 2268 开头 |
| PayerType | 付款方类型 | MERCHANT |
| PayeeId | 收款方 ID | 网商商户号（2268 开头）或 isvOrgId |
| PayeeType | 收款方类型 | MERCHANT / PLATFORM |

> **注意**：
> - 收款方只能为平台结算户 / 云资金企业商户 / 云资金个体工商户（个体工商户收款需申请加白）
> - 付款方商户授权状态必须为 VALID 才能发起代扣
> - 如接入资金冻结解冻控制方案，Scene 选择 `PROTOCOL_WITHHOLD_SUPPORT_FREEZE`，需指定 FreezeAmount 和 AvailableAmount（TotalAmount = FreezeAmount + AvailableAmount）

**代扣付款流水号前缀**：N

#### 代扣退款

协议代扣支持退款，通过协议代扣退款申请接口（`ant.mybank.bkcloudfunds.protocol.withhold.refund.apply`）发起，退款结果通过退款结果查询接口（`ant.mybank.bkcloudfunds.protocol.withhold.refund.query`）查询。

### 二、授权代付（加盟授权代付）

#### 业务场景（5 个子场景）

| 子场景 | 场景码 | 绑定上限 | 单笔限额 | 月度限额 | 典型用途 |
|--------|--------|---------|---------|---------|---------|
| 租金代付 | RENT | 5 个收款方 | ≤2.5 万 | ≤25 万 | 门店租金支付 |
| 水电物业代付 | FEE | 10 个收款方 | ≤2 万 | ≤10 万 | 水电费、物业费支付 |
| 合伙人分红 | PARTNER | 40 个收款方 | ≤10 万 | ≤40 万 | 合伙人利润分配 |
| 工资代发 | SALARY | 50 个收款方 | ≤2 万 | ≤25 万 | 员工工资发放 |
| 报销代发 | CLAIM | 200 个收款方 | ≤1 万 | ≤10 万 | 员工报销款发放 |

**限额说明**：
- **绑定上限**：每个子场景最多可绑定的收款方账户数量
- **单笔限额**：该子场景下单笔转账的最高金额
- **月度限额**：该子场景下所有收款方月度转账总和上限
- **重要**：报销代发子场景收款方最多不超过 200 个，且五个子场景的收款方总数合计不超过 200 个

**收款方银行卡限制**：
- 仅支持银联储蓄卡（仅中国大陆地区的储蓄卡）
- 不支持信用卡、境外银行卡
- 支付宝账号收款无此限制

**资金流向（过桥机制）**：

```
商户云资金子户 → 西进户（过桥账户）→ 外部银行卡/支付宝
```

- 提现至西进户和支付给外部收方是同一笔业务指令的连续子步骤，由网商后台自动调度执行
- 资金不会在西进户内停留，商户无法中途拦截资金
- 西进户 ≠ 结算户 ≠ 提现卡

#### 签约/修改流程（平台主动发起）

```
平台调用签约申请/协议变更申请接口
    (同步：ant.mybank.bkcloudfunds.protocol.sign.apply，≤50 个收方
     异步：ant.mybank.bkcloudfunds.protocol.sign.async.apply，>50 个收方)
    ↓
状态"已申请" → 网商返回确认链接(AuthSignUrl)
    ↓
"待商户确认" → 商户通过链接打开网商小程序完成授权
    ↓
"商户已确认" → 系统签署合约
    ↓
通过协议签约异步申请结果通知接口(ant.mybank.bkcloudfunds.protocol.sign.apply.notify)
    或查询申请单目前状态接口(ant.mybank.bkcloudfunds.protocol.sign.apply.query)获取结果
    ↓
"申请成功" → 平台获得合约号(AgreementNo)
"申请失败" → 记录失败原因
```

**签约接口核心入参**：

| 参数 | 说明 | 示例 |
|------|------|------|
| SceneCode | 业务场景 | PARTNER_CASCADE_PAY |
| OutBizNo | 外部流水号（幂等键） | - |
| AuthDetail | 授权事项（map JSON base64 编码） | 含 sceneCode、totalQuota、monthQuota、receiverList |

**AuthDetail 子场景码**：PARTNER / RENT / FEE / SALARY / CLAIM

**商户确认超时自动确认规则**：
- 5 个业务子场景中，**仅工资代发场景**支持超时自动确认
- 如果只修改工资场景的合约信息，商户 48 小时内未确认时，网商会自动确认通过
- 其他场景不支持超时自动确认，必须商户手动确认

#### 解约流程（无需平台审核）

```
商户在小程序发起解约
    ↓
网商通过授权合同变更通知接口(ant.mybank.bkcloudfunds.protocol.sign.notify)通知平台
    (AgreementStatus=INVALID)
    ↓
合约直接失效，平台无需审核
```

> 收到通知后应立即调用查询当前授权协议内容接口（`ant.mybank.bkcloudfunds.protocol.arr.query`）查询合约最新内容。

#### 代付付款流程

```
调用授权协议支付申请接口(ant.mybank.bkcloudfunds.protocol.pay.apply)
    → 状态"处理中"
    → 通过代付结果通知接口(ant.mybank.bkcloudfunds.trade.operate.result.notify)接收结果
       或调用代付结果查询接口(ant.mybank.bkcloudfunds.trade.operate.result.query)主动查询
    → "成功" 或 "失败"
```

**授权协议支付申请接口核心入参**：

| 参数 | 说明 | 示例 |
|------|------|------|
| SceneCode | 产品场景码 | PARTNER_CASCADE_PAY |
| SubSceneCode | 子场景码 | PARTNER / RENT / FEE / SALARY / CLAIM |
| AgreementNo | 协议支付合约号 | 签约接口返回的合约号 |
| TotalAmount | 支付金额（单位：分） | - |
| PayerParticipant | 付款方主体信息 | map JSON base64 编码 |
| PayeeParticipant | 收款方主体信息 | map JSON base64 编码 |

**代付付款流水号前缀**：W

**代付付款状态**：SUCCESS / FAIL / DEALING

### 三、签约/解约状态流转

#### 协议代扣（余额划转）签约状态

**申请单状态（4 种）**：

| 状态 | 说明 |
|------|------|
| 待签约 | 申请单已创建，等待商户签约 |
| 签约中 | 商户已打开签约链接，未完成 |
| 已签约 | 商户已完成签约 |
| 已解约 | 商户已解约 |

**合约状态（4 种）**：

| 状态 | 说明 |
|------|------|
| 生效中（VALID） | 合约正常，可发起代扣 |
| 已过期（UN_VALID） | 合约超过有效期 |
| 已解约 | 商户主动解约 |
| 已失效（INVALID） | 合约被平台或网商失效 |

**查询接口返回的状态枚举**：NOT_EXIST（不存在）、UN_VALID（已申请未生效）、VALID（已生效）、INVALID_TO_BE_CONFIRM（解约中待平台确认）

#### 授权代付签约状态

**申请单状态（6 种）**：

| 状态 | 说明 |
|------|------|
| INIT | 初始状态 |
| WAIT_CONFIRM | 待确认（平台已发起签约申请，等待商户确认） |
| CONFIRMED | 已确认（中间态，系统签署合约中） |
| SUCCESS | 成功（终态） |
| FAIL | 失败（终态） |
| CLOSED | 关闭（终态） |

**合约状态（3 种）**：

| 状态 | 说明 |
|------|------|
| 生效中（VALID） | 合约正常，可发起代付 |
| 已失效（INVALID） | 合约已失效 |
| 已取消（CANCELLED） | 合约已取消 |

## 业务规则

### 协议代扣 vs 授权代付核心差异

| 维度 | 协议代扣（余额划转） | 授权代付 |
|------|---------------------|---------|
| 业务用途 | 体系内子户间划转 | 对外付款（银行卡/支付宝） |
| 合同术语 | 余额划转 | 授权代付 |
| 驱动模式 | 事件驱动型 | 流程驱动型 |
| 签约发起方 | 商户（小程序主动签约） | 平台（主动发起签约申请） |
| 签约流程 | 商户签约 → 网商通知平台 | 平台申请 → 返回确认链接 → 商户确认 → 网商通知 |
| 解约发起方 | 商户（需平台审核） | 商户（无需审核） |
| 费率 | 0.00%（免费） | 0.05%（万五） |
| 收款方 ID | 网商商户号（2268 开头）或 isvOrgId | 外部银行卡号/支付宝账号 |
| 资金去向 | 8888 开头保证金户（结算户） | 外部银行账户/支付宝 |

### 术语对应

| 系统术语 | 合同术语 | 说明 |
|---------|---------|------|
| 协议代扣 | 余额划转 | 体系内子户间划转 |
| 加盟授权代付 | 授权代付 | 体系外向外部第三方付款 |
| 联动支付 | 协议代扣 | 同一体系内的子户间资金划转 |
| 授权代付 | 授权代付 | 向体系外付款 |

### 费率规则

| 收费项目 | 费率 | 收取方式 | 承担方 |
|---------|------|---------|--------|
| 协议代扣 | 0.00%（免费） | - | - |
| 授权代付 | 0.05%（万五） | D+1 从平台 8888 保证金户（结算户）自动扣除 | 平台（不建议转嫁给商户） |
| 提现 | 0.05%（万五） | D+1 从平台 8888 保证金户（结算户）自动扣除 | 商户（平台先行垫付，D+2 协议代扣向商户追缴） |

### 付款方式矩阵

| | 协议代扣（对内） | 授权代付（对外） |
|--|----------------|----------------|
| **批量付款** | 上传表格，多笔一批提交 | 上传表格，多笔一批提交 |
| **单笔付款** | 填写表单，逐笔提交 | 填写表单，逐笔提交 |

### 批量付款批次状态

| 状态 | 含义 |
|------|------|
| 处理中 | 系统解析中或网商处理中，仍有未返回结果的付款单 |
| 成功 | 批次内所有付款单全部成功 |
| 失败 | 全部解析失败或全部付款单失败 |
| 部分成功 | 部分成功部分失败 |

### 付款单状态

| 状态 | 说明 |
|------|------|
| 处理中 | 付款单已提交，网商正在处理（对应查询状态 DEALING） |
| 成功 | 付款成功（对应查询状态 SUCCESS） |
| 失败 | 付款失败（对应查询状态 FAIL） |
| 部分退款 | 付款成功后发生了部分退款 |
| 已退款 | 付款成功后已全额退款 |

### 最大可退金额计算

对于保证金充值场景：

- **ActualAmount**：充值保证金子户当前余额
- **maxRefundAmount**：每笔收方最大可退金额（随退回操作变化）
- **每笔实际可退金额** = min(ActualAmount, maxRefundAmount)

对于营销补贴分账退款场景：

- 退款金额不能超过原营销补贴分账金额

### 业务流水号规则

| 类型 | 规则 |
|------|------|
| 批量付款批次号 | 00120260525（3 位当日自增 + YYYYMMDD） |
| 批量付款单号 | 001_00120260528（单内序号 + 批次号） |
| 单笔付款单号 | 001_50120260528（单内序号 + 501 起始的自增 + YYYYMMDD） |

## 注意事项

1. **协议代扣前必须确认合约状态**：发起代扣前必须查询确认付款方商户的授权状态为 VALID，否则接口会报错。

2. **授权代付手续费不建议转嫁**：对外支付需保证金额精确，转嫁手续费会导致收款方收到金额减少，影响体验。

3. **授权代付需先签约后付款**：签约流程中商户必须通过确认链接完成授权，平台无法代替商户确认。

4. **取消申请单**：在申请单被商户确认之前，平台无法再次发起申请。如需修改，需先调用取消申请接口（`ant.mybank.bkcloudfunds.protocol.sign.apply.cancel`）取消后再重新发起。

5. **通知乱序处理**：授权合同变更通知可能乱序到达，建议根据 AgreementNo 反查当前生效的合约，而不是依赖通知中的状态。

6. **个体工商户加白**：协议代扣中，个体工商户作为收款方需要申请加白。操作路径：连锁通管理中心登录 → 业务变更申请 → 新增业务申请 → 业务限制变更类型 → 个体工商户支持协议代扣。付款方不需要加白。

7. **子户对外月度转账限额**：约 40 万上限。如需突破，可在同一营业执照下注册多个商户号。

8. **提现手续费收取流程**：提现手续费由商户承担但商户提现时不感知（全额到账）。平台 D+1 先行垫付，D+2 凌晨 3 点拉取品牌方费用对账文件（`ant.mybank.bkmbp.stmt.fee.file.apply`），与本地提现订单比对后通过协议代扣向商户逐笔追缴。

## 常见误区

1. **误区：协议代扣和授权代付的签约方式相同**
   实际：协议代扣由商户在小程序主动发起签约；授权代付由平台主动发起签约申请，商户通过确认链接完成授权。两者的签约发起方完全不同。

2. **误区：协议代扣解约无需审核**
   实际：协议代扣解约需要平台审核（AGREE/REJECT）；授权代付解约无需审核，商户发起后合约直接失效。

3. **误区：授权代付资金直接从商户子户到外部银行卡**
   实际：资金经过西进户（过桥账户）中转，但这是同一笔业务指令的连续子步骤，由网商后台自动调度，资金不会在西进户停留，商户无法中途拦截。

4. **误区：所有授权代付子场景都支持商户确认超时自动确认**
   实际：仅工资代发（SALARY）场景支持 48 小时超时自动确认，其他 4 个子场景必须商户手动确认。

5. **误区：授权代付手续费由商户承担**
   实际：授权代付手续费由平台承担（不建议转嫁），因为转嫁会导致收款方收到金额减少。提现手续费才由商户承担。

6. **误区：联动支付和授权代付是同一种付款方式**
   实际：联动支付是协议代扣的别称（体系内子户间划转，免费）；授权代付是向体系外付款（收费 0.05%）。两者本质不同。

7. **误区：保证金户充值后 maxRefundAmount 等于充值金额**
   实际：每笔实际可退金额取 ActualAmount（保证金子户当前余额）和 maxRefundAmount（每笔最大可退金额，随退回操作变化）的最小值，不一定等于原始充值金额。

8. **误区：收到授权合同变更通知后可以直接使用通知中的合约状态**
   实际：通知可能乱序到达，建议收到通知后主动调用查询当前授权协议内容接口确认合约最新状态。

9. **误区：协议代扣的收款方可以是任意商户**
   实际：收款方只能为平台结算户（8888 开头）、云资金企业商户或云资金个体工商户，且个体工商户收款需申请加白。
