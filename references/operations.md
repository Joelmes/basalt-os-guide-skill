# 01.1_总部公司/区域公司入驻操作手册

---

## 一、概述

商户入驻是 Basalt OS 清算分账体系的第一步。商户通过网商银行小程序完成开户后，才能在系统中进行清算、付款、提现等资金操作。

当前商户入驻方式为**小程序入驻**：由平台运营人员在 Basalt OS 中创建商户主体并生成注册码，商户法人扫码进入网商银行小程序自行完成入驻。后续版本将增加"预入驻+激活"方式。

本手册适用于**总部公司**和**区域公司**的入驻操作。

> ⚠️ **重要提醒：商户入驻只能由营业执照上的法人使用本人支付宝操作，不可由他人代办。**

---

## 二、创建商户主体

在生成注册码之前，需要先在系统中创建好对应的公司。

### 2.1 添加总部公司

操作路径：清算分账 → 总部公司 → 总部公司→ 点击「添加」按钮

填写以下信息：

- **业务属性**（必选）：从「平台管理 - 业务属性」中已启用的业务属性中选择。
- **公司名称**（必填）：文本输入。
- **统一社会信用代码**（选填）：文本输入。

### 2.2 添加区域公司

操作路径：清算分账 → 区域公司 → 区域公司列表 → 点击「添加」按钮

填写以下信息：

- **所属区域**（必选）：从「平台管理 - 区域划分」中已启用的区域中选择。
- **业务属性**（必选）：从「平台管理 - 业务属性」中已启用的业务属性中选择。
- **公司性质**（单选）：

  - 选择"区域总公司"：该公司为所选区域的总公司。
  - 选择"区域业务子公司"：需额外选择一家同区域的区域总公司作为上级。
- **公司名称**（必填）：文本输入。
- **统一社会信用代码**（选填）：文本输入。

填写完成后点击「保存」，系统将按编码规则自动生成唯一的**公司编码**。

【待补充：区域公司法人在支付宝小程序中的具体操作和填写内容】

## 三、生成商户注册码

注册码是商户入驻的唯一凭证，每个注册码对应一个独立的外部商户号。

**重要提醒：每个商户的注册码唯一，请勿共用或串用。**

### 3.1 单个生成

在总部公司/区域公司列表中，点击对应主体操作列的「注册码」按钮，进入注册码管理面板。

面板中展示以下信息：

- **主体名称**：公司名称（自动带入）。
- **主体编码**：公司编码（自动带入）。
- **主体属性**：公司属性（自动带入）。

点击「新增注册码」按钮，系统将：

1. 按编码规则生成一个唯一的**外部商户号**。
2. 按网商银行规范生成**注册链接**。
3. 将注册链接生成**注册二维码**。

**注意：新增注册码操作不可撤销。** 点击后立即新增一条关联商户数据，并在商户列表顶部出现。注册码不支持删除和修改。一个主体可以新增多个注册码。

### 3.2 批量生成

在区域公司列表中，勾选多个主体后点击「（批量）生成注册码」按钮，弹窗提示如下：

- 若主体暂无注册码，将为其生成新注册码。
- 若主体已有注册码，则将为其新增一个新注册码。

提供两种操作：

- **仅生成**：仅生成注册码，不下载，异步完成。
- **生成并下载**：异步生成商户入驻注册二维码图片，图片以"主体名称+外部商户号"命名，格式为 .png，打包成 .zip 文件通过浏览器自动下载。

### 3.3 注册码信息说明

每个注册码包含以下可用操作：

| 操作 | 说明 |
|-|-|
| 复制 | 将完整的注册链接复制到剪贴板，可直接发送给商户 |
| 下载 | 将注册链接生成二维码图片并下载至本地 |

生成注册码后，对应的商户数据会自动出现在商户列表中，初始入驻状态为「未入驻」。

## 四、商户扫码入驻

### 4.1 入驻操作

将注册码的`二维码图片`或`注册链接`通过`钉钉`分发给对应的商户法人。商户法人使用支付宝扫描二维码（或在钉钉内打开注册链接），进入网商银行云资金小程序，按指引自行完成入驻流程。

商户入驻需要填写以下关键信息

1. **选择企业证件类型**：选「营业执照」。
2. **营业执照信息**：

   1. 下拉切换到「公司名称」，然后输入完整的公司名称，需与营业执照一致。
   2. 上传营业执照图片。如果公司已经在网商银行做过认证则系统会自己把营业执照信息带出来。上传之后核对营业执照信息。
   3. 确认主体信息：上传法人身份证正反面照片。上传之后核对法人身份证信息。
3. **商户申请信息**：

   - 商户角色：先左边上滑选择「品牌总部」，然后根据实际情况选择右侧的内容。
   
     - 品牌授权及公司：选「品牌管理公司」。
     - 为门店提供其他服务的公司：选「其他服务公司」。
     - 区域公司：选「品牌区域公司」
   - 经办人手机号、联系人、商户联系电话：填写法人信息。
   - 所在地区、详细地址：跟营业执照一致的地址
4. **填写企业对公账户**：银行账号和收款银行（开户行），开户地区，开户支行，需与对公账户的开户信息一致。
5. **提交申请验证**（小额打款验证）：网商银行会上一步填写的银行卡转入小金额，填写该金额数字。
6. **身份验证**：操作人本人支付宝支付密码。

入驻过程中，商户需要完成：

- 营业执照信息录入
- 法人身份认证和资质审核
- 结算银行卡绑定
- 签署相关协议

【待补充：总部公司/区域公司法人在支付宝小程序中的具体填写内容和操作步骤】

### 4.2 入驻状态说明

商户入驻过程中，系统中的入驻状态会随网商银行接口返回值自动更新。各状态含义如下：

| 入驻状态 | 含义 |
|-|-|
| 未入驻 | 注册码刚创建，商户尚未开始入驻 |
| 预入驻审核中 | （预入驻方式）商户预入驻申请已提交，等待网商审核 |
| 待激活 | （预入驻方式）预入驻审核通过，等待商户继续激活 |
| 预入驻失败 | （预入驻方式）预入驻审核未通过，可查看失败原因 |
| 激活中 | 商户正在进行入驻/激活操作，网商处理中 |
| 激活成功 | 商户入驻完成，可正常使用全部功能 |
| 激活失败 | 商户激活未通过，可查看失败原因 |

### 4.3 查看与更新入驻状态

操作路径：清算分账 → 对应主体类型的商户列表

入驻状态不会自动实时刷新，运营人员需要手动更新：

1. 在商户列表中勾选需要更新的商户。
2. 点击「更新入驻状态」按钮。
3. 系统调用网商银行接口查询最新入驻状态并更新。

如果入驻失败，状态文字后方会显示提示符号，点击可打开弹窗查看完整的失败原因。

## 五、入驻后配置

商户激活成功后，还需要完成以下配置才能正常使用清算分账功能。

### 5.1 更新商户信息

操作：在商户列表中勾选商户 → 点击「更新商户信息」

系统将依次执行：

1. 调用网商银行接口获取商户在网商侧的最新商户信息（异步执行）。
2. 调用商户子户查询接口查询商户保证金户数据。

### 5.2 开通保证金户（待上线）

操作路径：商户详情页 → 资金账户 → 账户信息区域

如果商户尚未开通保证金户，页面会显示「未开通」及「开通」按钮。点击「开通」后，系统调用商户开户接口为该商户开通保证金子户（TRADE_DEPOSIT），同步返回保证金户账号。

### 5.3 签约授权

商户入驻成功后，需要根据业务需要完成签约授权，包括两种合约：

**协议代扣**：由商户在支付宝云资金小程序中主动发起签约，网商通过通知接口将合约号返回平台。

**授权代付**：由平台运营人员发起。操作路径：签约授权 → 新增/修改授权代付合约。

发起授权代付的前置条件：商户已在网商云资金小程序中自行开通**过桥西进户**（网商不会通知也无法查询是否已开通）。

授权代付合约需要填写：

- **授权方**：选择公司和对应商户。系统会自动查询该商户是否已有生效合约，有则走修改逻辑，无则走新增逻辑。
- **备注**：不超过512个字符。
- **动账短信**：选择是否发送动账短信通知。
- **收款方信息**：按五个业务子场景分别填写收款方信息。

五个业务子场景及其限制：

| 场景 | 场景码 | 单笔上限 | 月度上限 | 收款方数量上限 |
|-|-|-|-|-|
| 租金 | RENT | 2.5万 | 25万 | 5个 |
| 水电物业费 | FEE | 2万 | 10万 | 10个 |
| 合伙人 | PARTNER | 10万 | 40万 | 40个 |
| 工资 | SALARY | 2万 | 25万 | 50个 |
| 报销代发 | CLAIM | 1万 | 10万 | 500个 |

五个场景所有收款方总计最多可绑定 200 个授权方。收款方支持银行卡（对公/对私银联借记卡，不支持信用卡）和支付宝（仅支持个人对私账户）。

填写完成后点击「提交」，系统校验数据有效性后调用网商银行接口发起授权申请。收款方数量不超过50个走同步接口，超过50个走异步接口。

## 七、常见问题与注意事项

**Q：注册码生成后可以删除或修改吗？**  
A：不可以。注册码一旦新增即不可撤销，不支持删除和修改操作。请在分发前确认信息准确。

**Q：一个公司可以有几个商户？**  
A：一个主体可以新增多个注册码，每个注册码对应一个独立的商户。当主体关联多个商户时，列表默认展示最早创建的商户信息，并显示"等 x 个商户"的链接，点击可查看全部。

**Q：商户入驻失败怎么办？**  
A：在商户列表中，入驻失败的状态后方会显示提示符号，点击可查看网商银行返回的完整失败原因。根据失败原因修正后，商户可重新扫码入驻。

**Q：授权代付合约的过桥西进户在哪里开通？**  
A：过桥西进户由商户在网商云资金小程序中自行开通，品牌方无法代为操作，网商也不会通知是否已开通。在发起授权代付申请前，务必确认商户已完成此步骤。

**Q：签约授权的收款方信息填写有哪些限制？**  
A：银行卡仅支持银联借记卡（对公/对私），不支持信用卡；支付宝仅支持个人对私账户，不支持对公类型。对公银行卡需填写联行号。五个场景的收款方总数不超过200个。

## 四、商户扫码入驻

### 4.1 入驻操作

将注册码的`二维码图片`或`注册链接`通过`钉钉`分发给对应的门店法人。门店法人使用支付宝扫描二维码（或在钉钉内打开注册链接），进入网商银行云资金小程序，按指引自行完成入驻流程。

门店入驻需填写以下关键信息：

1. **商户类型**：按营业执照，目前门店都是个体工商户。
2. **选择门店信息**：选择跟营业执照一致的企业名称。
3. **商户申请信息页面**：

   - 商户角色：选「门店-联营门店」
   - 经营者手机号、联系人、商户联系电话：填写法人/负责人信息
   - 所在地区、详细地址：跟营业执照一致的地址
4. **绑定收款账户**：可以先选择支付宝，后面提现时再绑定银行卡也可以。如果绑定银行卡则只能选择法人/负责人本人的银行卡。
5. **身份验证**：操作人本人支付宝支付密码。

### 4.2 入驻状态说明

商户入驻过程中，系统中的入驻状态会随网商银行接口返回值自动更新。各状态含义如下：

| 入驻状态 | 含义 |
|-|-|
| 未入驻 | 注册码刚创建，商户尚未开始入驻 |
| 预入驻审核中 | （预入驻方式）商户预入驻申请已提交，等待网商审核 |
| 待激活 | （预入驻方式）预入驻审核通过，等待商户继续激活 |
| 预入驻失败 | （预入驻方式）预入驻审核未通过，可查看失败原因 |
| 激活中 | 商户正在进行入驻/激活操作，网商处理中 |
| 激活成功 | 商户入驻完成，可正常使用全部功能 |
| 激活失败 | 商户激活未通过，可查看失败原因 |

### 4.3 查看与更新入驻状态

操作路径：清算分账 → 门店 → 门店商户

入驻状态不会自动实时刷新，运营人员需要手动更新：

1. 在商户列表中勾选需要更新的商户。
2. 点击「更新入驻状态」按钮。
3. 系统调用网商银行接口查询最新入驻状态并更新。

如果入驻失败，状态文字后方会显示提示符号，点击可打开弹窗查看完整的失败原因。

## 五、入驻后配置

商户激活成功后，还需要完成以下配置才能正常使用清算分账功能。

### 5.1 更新商户信息

操作：在商户列表中勾选商户 → 点击「更新商户信息」

系统将依次执行：

1. 调用网商银行接口获取商户在网商侧的最新商户信息（异步执行）。
2. 调用商户子户查询接口查询商户保证金户数据。

### 5.2 开通保证金户（待上线）

操作路径：商户详情页 → 资金账户 → 账户信息区域

如果商户尚未开通保证金户，页面会显示「未开通」及「开通」按钮。点击「开通」后，系统调用商户开户接口为该商户开通保证金子户（TRADE_DEPOSIT），同步返回保证金户账号。

### 5.3 签约授权

商户入驻成功后，需要根据业务需要完成签约授权，包括两种合约：

**协议代扣**：由商户在支付宝云资金小程序中主动发起签约，网商通过通知接口将合约号返回平台。

**授权代付**：由平台运营人员发起。操作路径：签约授权 → 新增/修改授权代付合约。

发起授权代付的前置条件：商户已在网商云资金小程序中自行开通**过桥西进户**（网商不会通知也无法查询是否已开通）。

授权代付合约需要填写：

- **授权方**：选择门店和对应商户。系统会自动查询该商户是否已有生效合约，有则走修改逻辑，无则走新增逻辑。
- **备注**：不超过512个字符。
- **动账短信**：选择是否发送动账短信通知。
- **收款方信息**：按五个业务子场景分别填写收款方信息。

五个业务子场景及其限制：

| 场景 | 场景码 | 单笔上限 | 月度上限 | 收款方数量上限 |
|-|-|-|-|-|
| 租金 | RENT | 2.5万 | 25万 | 5个 |
| 水电物业费 | FEE | 2万 | 10万 | 10个 |
| 合伙人 | PARTNER | 10万 | 40万 | 40个 |
| 工资 | SALARY | 2万 | 25万 | 50个 |
| 报销代发 | CLAIM | 1万 | 10万 | 500个 |

五个场景所有收款方总计最多可绑定 200 个授权方。收款方支持银行卡（对公/对私银联借记卡，不支持信用卡）和支付宝（仅支持个人对私账户）。

填写完成后点击「提交」，系统校验数据有效性后调用网商银行接口发起授权申请。收款方数量不超过50个走同步接口，超过50个走异步接口。

## 七、常见问题与注意事项

**Q：注册码生成后可以删除或修改吗？**  
A：不可以。注册码一旦新增即不可撤销，不支持删除和修改操作。请在分发前确认信息准确。

**Q：一个门店可以有几个商户？**  
A：一个主体可以新增多个注册码，每个注册码对应一个独立的商户。当主体关联多个商户时，列表默认展示最早创建的商户信息，并显示"等 x 个商户"的链接，点击可查看全部。

**Q：商户入驻失败怎么办？**  
A：在商户列表中，入驻失败的状态后方会显示提示符号，点击可查看网商银行返回的完整失败原因。根据失败原因修正后，商户可重新扫码入驻。

**Q：授权代付合约的过桥西进户在哪里开通？**  
A：过桥西进户由商户在网商云资金小程序中自行开通，品牌方无法代为操作，网商也不会通知是否已开通。在发起授权代付申请前，务必确认商户已完成此步骤。

**Q：签约授权的收款方信息填写有哪些限制？**  
A：银行卡仅支持银联借记卡（对公/对私），不支持信用卡；支付宝仅支持个人对私账户，不支持对公类型。对公银行卡需填写联行号。五个场景的收款方总数不超过200个。

<title>02_清算操作手册</title>

# 一、概述

清算是 Basalt OS 的核心资金链路，负责将门店的收银资金通过网商银行完成归集、确认和分配。完整流程如下：

```
收钱吧 POS 收款（资金落在华夏银行卡）→ 订单同步（D-1，收钱吧推送）→ 订单上送（前置：POS 绑定商户）→ 生成清算文件 → 资金转入平台清算专户（9903）→ 批次上报 → 扣保退保（如有）
```

<callout emoji="💡">
其中"收钱吧 POS 收款"和"资金转入平台清算专户"是线下步骤。系统仅处理信息流（订单数据自动同步），资金流需要人工操作。
</callout>

当前订单来源为收钱吧（POS 收银订单），三联 ERP 清算订单通道尚未上线。

---

# 二、收钱吧 POS 收款（线下步骤）

<callout emoji="📌">
本步骤为线下操作，不涉及 Basalt OS 系统。
</callout>

门店通过收钱吧 POS 机完成收银后，资金会进入 POS 机绑定的华夏银行卡中。这一步是整个清算链路的起点，资金暂时停留在华夏银行卡，后续需要手动转入网商清算专户（详见第七章）。

**要点：**

- POS 机与华夏银行卡的绑定关系，由收钱吧侧管理。
- 收钱吧对账单中仅显示 POS 机名称，不显示门店或商户信息，因此需要在系统中将 POS 机绑定到对应商户（详见第四章前置配置）。

# 三、订单同步（D-1，收钱吧推送）

收钱吧每天自动将 D-1（前一天）的账单数据推送至可柔 FTP 服务器，Basalt OS 自动解析并入库。系统通过POS机与商户的绑定关系（详见第四章前置配置）将订单分配到对应门店。

操作路径：清算分账 → 清算订单（收钱吧）

## 3.1 拉取方式

- **自动拉取**：收钱吧每天推送一次，包含所有门店的收钱吧订单。
- **手动拉取**：无手动拉取入口。

## 3.2 订单列表字段

| 字段 | 说明 |
|-|-|
| 订单编号 | 收钱吧对账单中的商户订单号 |
| 实收金额（元） | 收钱吧对账单中的收款金额 |
| 付款方式 | 收钱吧对账单中的收款通道/支付方式 |
| 渠道类型 | 0：门店订单（当前全部），1：微信小程序订单（预留） |
| 门店名称 | 通过 POS 机名称匹配三联系统中的门店名称 |
| 支付时间 | 收钱吧对账单中的交易日期+时间 |
| 清算状态 | 订单在清算流程中的当前状态 |
| 上送失败原因 | 上送失败时展示，包含网商侧和本系统侧的失败原因 |
| 清算失败原因 | 清算失败时展示，包含网商侧和本系统侧的失败原因 |
| 网商订单号 | 上送成功后网商返回，未上送订单无此值 |
| 批次号 | 生成清算文件后产生 |

# 四、订单上送

订单上送是将拉取到的收银订单数据提交给网商银行进行确认的过程。上送成功后，订单才具备进入清算的资格。

操作路径：清算分账 → 清算订单（收钱吧）→ 筛选「未上送」状态 → 勾选订单 → 点击「上送订单」

<callout emoji="❗">
**前置条件：POS 必须已绑定商户。**收钱吧账单仅显示 POS 机名称，需要将 POS 绑定至门店商户后才能执行上送操作。绑定方式详见下方 4.1 节。
</callout>

## 4.1 前置配置——POS 绑定商户

操作路径：清算分账 → 清算订单→ POS 设置

- POS 机数据从收钱吧推送的对账单中自动获取，不支持手动添加。
- POS 与商户为多对一关系：一个 POS 只能绑定至一个商户，一个商户可以被多个 POS 绑定。
- 绑定方式：

  - 单个绑定：点击POS名称后面的「绑定」→ 在弹窗中通过商户名称或门店名称筛选目标商户，选中后点击确定按钮完成绑定，绑定成功后可在订单列表中通过门店名称筛选订单。
  - 点击「绑定 POS」→ 下载空白模板 → 填写 POS 机名称（须与收钱吧对账单严格一致）、商户简称、门店名称 → 上传 Excel 文件 → 系统异步解析完成绑定。绑定成功后可在订单列表中通过门店名称筛选订单。
- 解绑 POS ：由于解绑操作会影响订单和门店的归属关系，目前暂未提供解绑功能，如发现绑定错误确需解绑的，请联系郭玉卓通过技术手段解绑。

## 4.2 上送操作步骤

1. 在订单列表中筛选状态为「未上送」的订单。
2. 勾选需要上送的订单。
3. 点击「上送订单」按钮。

## 4.3 系统校验与处理

**校验一：未勾选订单**  
提示「请先选择未上送的订单，然后再执行当前操作」。

**校验二：POS 未绑定商户**  
如勾选订单中包含所属 POS 机未绑定商户的订单：弹窗提示，可选择「重新选择」或「继续上送」（系统自动过滤未绑定商户的订单，仅对已绑定商户的订单执行上送）。

**校验三：包含非"未上送"状态的订单**  
弹窗提示，可选择「重新选择」或「继续上送」（系统自动过滤已上送订单，仅对未上送订单执行操作）。

**校验四：勾选无异常**  
系统依次执行：支付单调用网商补单创单接口执行上送；退款单仅变更状态为「已上送」，不调用网商接口。弹窗提示「订单上送将在后台执行，稍后可刷新列表查看上送状态」。

## 4.4 上送结果

- **上送成功**：订单状态变为「已上送」，网商返回网商订单号。提示「订单上送成功，请刷新列表查看」。
- **上送失败**：订单状态变为「上送失败」，列表显示失败原因（含网商侧和本系统侧的失败原因）。可对「上送失败」状态的订单重新执行上送操作。

# 五、生成清算文件

清算文件（网商有时候也称"补单文件"或"交易明细文件"）承载本次参与清算的商户及交易订单信息。网商银行根据该文件中的商户维度进行资金划转。

<callout emoji="💡">
每天最多上传 50 次清算文件。
</callout>

## 5.1 操作步骤

1. 在清算订单列表中筛选状态为「已上送」且批次号为"-"的订单。
2. 勾选需要参与本次清算的订单。
3. 点击「生成清算文件」按钮。

## 5.2 系统校验

**情况一：勾选中包含"未上送"订单**  
系统弹窗提示，选择「继续生成」后将自动对未上送订单执行上送操作，上送成功后再生成清算文件。上送失败的订单会被自动过滤，不纳入清算文件。

**情况二：勾选中包含已有批次号的订单**  
系统弹窗提示，选择「继续生成」后将自动过滤已有批次号的订单，仅对批次号为"-"的订单生成清算文件。

**情况三：两种异常同时存在**  
系统弹窗同时提示，选择「继续生成」后先执行上送操作（过滤上送失败的），再过滤已有批次号的，仅对合格的订单生成清算文件。

**情况四：勾选无异常**  
弹窗提示「清算文件将在后台生成，稍后可前往「清算文件」查看」。

## 5.3 清算文件说明

- 清算文件生成后系统自动生成批次号，清算文件以批次号命名。
- 清算文件管理页可查看该批次的实际清算金额，后续提交清算批次正式发起清算时，需确保品牌专户中的「待清算金额」大于待清算批次的实际金额。

## 5.4 清算文件管理

操作路径：清算分账 → 清算文件

| 字段 | 说明 |
|-|-|
| 批次号 | 系统生成的清算批次号，与文件名对应 |
| 交易总金额 | 支付订单总额 - 退款订单总额（= 实际金额 + 渠道手续费） |
| 实际金额 | 支付订单实收 - 退款订单实退 |
| 渠道手续费 | 支付渠道费 - 退款渠道退费 |
| 状态 | 清算文件的当前状态 |

**清算文件可用操作（按状态）：**

| 状态 | 可用操作 |
|-|-|
| 待上传 | 上传、作废、详情 |
| 上传失败 | 上传、作废、详情 |
| 已上传 | 撤回、新建批次、详情 |
| 已撤回 | 上传、作废、详情 |
| 待上报 | 查看批次、详情 |
| 上报失败 | 查看批次、详情 |
| 清算中 | 查看批次、详情 |
| 清算成功 | 查看批次、详情 |
| 部分成功 | 查看批次、详情 |
| 清算失败 | 查看批次、详情 |
| 已作废 | 详情 |

**关键操作说明：**

- **上传**：系统上传清算文件至网商银行的FTP服务器。
- **撤回**：从网商 FTP 服务器撤回已上传的文件。
- **作废**（不可撤销）：作废后该文件不可再编辑或上传，批次内所有订单恢复为"已上送"状态，需要重新生成清算文件。
- **新建批次**：上传成功后可从此处进入新建批次页面。

## 6.2 操作要点

- **金额要求**：需确保清算专户的待清算余额大于即将上报的清算批次对应文件中的实际金额。余额不足时，批次上报后清算会失败。
- **时机**：这一步可以提前做。建议在订单上送完成后即可安排转账，不必等到清算文件生成后。只要在批次上报前确保余额充足即可。
- **查看余额**：可在系统的「品牌专户」页面查看平台清算专户的待清算余额和今日入账金额。

# 七、批次上报

清算文件上传成功后，需要创建清算批次正式向网商银行发起清算。网商银行接收批次后，按商户维度聚合订单并进行资金划转。

## 7.1 新建批次

操作路径：清算分账 → 清算文件 → 状态为"已上传"的文件 → 点击「新建批次」

新建批次页面字段：

- **批次号**：从清算文件自动带入。
- **渠道编号**：从清算文件带入，当前统一为 DEFAULT。
- **清算类型**：从清算文件带入，固定为 STMT_AVAILABLE_SUM_SHARE（ISV 收单，结算至余额并分账）。
- **清算文件**：从清算文件带入，与批次号同名。
- **备注**（选填）：可填写本次批次的备注内容。

提交方式：

- **仅保存**：仅在本系统保存，不调用网商接口。批次状态为「待上报」。保存后可二次编辑或上报。
- **保存并上报**：保存后立即调用网商创建批次接口发起清算。上报成功则批次状态变为「清算中」，上报失败则变为「上报失败」。

## 7.2 批次管理

操作路径：清算分账 → 清算 → 清算批次

**筛选维度：** 上报时间、完成时间、批次实例号、批次号、是否扣保、批次状态。

**批量操作——更新批次状态：**

- 勾选指定批次后点击「更新批次状态」：查询勾选批次的最新状态。
- 不勾选直接点击：自动查询当前列表中所有「清算中」状态批次的最新状态。

**批次状态流转：**

| 状态 | 含义 | 可用操作 |
|-|-|-|
| 待上报 | 批次已保存，尚未上报 | 上报、作废、编辑 |
| 上报失败 | 调用创建批次接口失败 | 上报、作废、编辑 |
| 清算中 | 网商已接收，正在处理 | 详情 |
| 清算成功 | 全部订单清算完成 | 详情 |
| 部分成功 | 部分订单清算成功 | 详情 |
| 清算失败 | 清算未通过 | 详情 |
| 已作废 | 批次已作废 | 详情 |

**作废批次**（不可撤销）：作废后该批次不可再上报，清算文件同步作废，批次内所有订单恢复至"已上送"状态，需要重新生成清算文件后发起清算。

## 7.3 清算批次详情

批次详情页展示完整的清算信息：批次号、渠道编号、清算类型、清算文件名、批次实例号、状态、备注、上报时间、完成时间、结算单扣保信息（无扣保显示"0"，有扣保显示"共 x 个商户"）、失败原因（清算失败时显示错误描述，部分成功时显示失败信息下载地址）。

# 八、扣保退保（如有）

## 8.1 什么是扣保

清算批次提交后，网商银行按商户维度聚合订单进行清算。如果某商户存在退款订单，且该商户的可用余额不足以覆盖退款金额时，网商会从品牌方 888 结算户（平台保证金户）帮商户垫付该笔退款，这就是"扣保"。

扣保发生后：

- 清算批次结果中会显示扣保商户数量， IsDepositPay（是否发生结算单扣保）和 DepositPayCnt（发生扣保的商户数量）字段。
- 网商会通过补单保证金扣除通知接口（Type = PAY_FOR_STMT_NOT_ENOUGH）向品牌方发送扣保通知，每个被扣保的商户一条通知。

## 8.2 扣保记录管理

操作路径：清算分账 → 清算 → 退款扣保 → 扣保记录

系统收到扣保通知后自动创建扣保单，每条扣保单一一对应一个商户在某批次中的扣保事件。

| 字段 | 说明 |
|-|-|
| 批次号 | 发生扣保的清算批次号 |
| 商户 | 被扣保的商户（显示公司/门店名称和商户简称） |
| 扣保单号 | 网商返回的扣保单据号 |
| 金额 | 扣保金额 |
| 退保流水号 | 发起退保申请后产生，待发起状态时显示"-" |
| 网商操作单号 | 退保申请受理后网商返回 |
| 退保状态 | 扣保单的当前状态 |
| 失败原因 | 退保失败时显示 |

**批量操作——更新退保状态：**

- 未勾选时：查询所有「处理中」状态的退保单的最新状态。
- 勾选时：仅查询勾选范围内「处理中」状态的退保单。

## 8.3 退保操作

退保是将品牌方 888 结算户被垫付的资金从商户可用余额中归还的过程。必须在商户余额充足的情况下发起。

**操作步骤：**

1. 在扣保记录列表中找到状态为「待发起」的扣保单。
2. 点击操作列的「退保」按钮。
3. 系统调用网商结算单退保申请接口发起退保申请。

**退保流程：**

```
收到扣保通知 → 系统自动创建扣保单（待发起）→ 财务发起退保申请（处理中）→ 退保成功/退保失败
```

- 退保申请提交后，申请结果同步返回，退保结果异步返回。
- 退保成功后，网商会通过补单保证金退回通知接口（Type = RETURN_PLATFORM_DEPOSIT）发送退保成功通知。
- 退保失败时，列表显示网商返回的错误描述。

**退保状态：**

| 状态 | 含义 |
|-|-|
| 待发起 | 扣保已记录，尚未发起退保申请 |
| 处理中 | 退保申请已提交，等待网商处理 |
| 退保成功 | 资金已从商户余额退回品牌方 888 结算户 |
| 退保失败 | 退保未通过，可查看失败原因 |

## 8.4 扣保单详情

点击扣保单列表操作区的「详情」按钮，可查看完整的扣保和退保信息：批次号、商户、扣保单号、金额、退保流水号、网商操作单号、退保状态、错误描述（仅退保失败时可见）。

# 十、常见问题与注意事项

**Q：上送失败的订单怎么处理？**  
A：上送失败的订单状态变为"上送失败"，列表中会显示失败原因。可以点击该订单操作列的"上送"按钮重新执行上送。也可以在批量上送时一并勾选，系统会自动处理。

**Q：清算文件作废后订单怎么办？**  
A：清算文件作废后（不可撤销），文件内所有订单会恢复为"已上送"状态，可以重新勾选这些订单生成新的清算文件。

**Q：批次上报后可以撤回吗？**  
A：清算文件在"已上传"状态时可以撤回。但如果批次已经上报至网商（状态为"清算中"及之后），则无法撤回，只能等待清算结果。如果批次还在"待上报"状态，可以通过作废操作取消。

**Q：每天清算文件上传次数有上限吗？**  
A：每天最多上传 50 次清算文件，请合理规划每批次的订单数量。

**Q：扣保发生后必须手动退保吗？**  
A：是的。收到扣保通知后系统会自动创建扣保单，但退保需要运营人员手动发起。退保前请确认商户可用余额充足，否则退保可能失败。

**Q：清算批次中部分成功是什么意思？**  
A：部分成功表示该批次中部分商户的清算通过、部分未通过。网商会返回失败信息文件的下载地址（三天有效），系统会自动下载并展示失败原因。可以根据失败原因对问题订单重新处理。

**Q：上报清算批次前需要注意什么资金条件？**  
A：在正式上报清算批次前，财务人员需要手动将 POS 收银资金从华夏银行卡转入网商平台清算专户（9903 开头），并确保清算专户的待清算余额大于清算批次对应文件中的实际金额。如果余额不足，批次上报后清算会失败。

**Q：资金转入清算专户可以提前做吗？**  
A：可以。这一步不必等到清算文件生成后再做，建议在订单上送完成后即可安排转账，只要在批次上报前确保余额充足即可。

**Q：清算订单（三联）通道现在可以使用吗？**  
A：不可以。三联 ERP 清算订单通道目前标注为"未上线"，当前实际订单来源全部为收钱吧 FTP 推送。三联通道为预留设计，启用时间待定。
