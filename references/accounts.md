# Basalt OS 账户体系参考手册

## 概述

Basalt OS 清算分账系统基于网商银行云资金平台，采用多账户体系管理资金流转。系统涉及5类账户，分别承担资金归集、商户结算、平台收费等不同职能。所有账户均由网商银行分配和管理，平台系统通过接口进行查询和操作。

## 账户类型定义

### 1. 平台清算专户（9903 开头）

| 属性 | 说明 |
|------|------|
| 账号规则 | 9903 开头 |
| 合同术语 | 交易见证账款管理专用编号 / 平台交易资金汇总专管户 |
| 当前账号 | 99030360620900482009 |
| 户名 | 浙江可柔品牌管理有限公司 |
| 开户行 | 浙江网商银行 |
| 用途 | 接收品牌方每日手动汇款，再通过清算批次划拨至各商户子户 |
| 开通方式 | 对接清算接口后由网商银行分配 |

### 2. 平台保证金户/结算户（8888 开头）

| 属性 | 说明 |
|------|------|
| 账号规则 | 8888 开头 |
| 当前账号 | 8888888815513823 |
| 用途 | 平台自有资金储值；接收协议代扣（余额划转）款项；提现手续费和授权代付手续费由网商 D+1 从此户自动扣除 |
| 开通方式 | 通过支付宝 → 网商银行小程序完成储值 |

### 3. 商户子户（2268 开头）

| 属性 | 说明 |
|------|------|
| 账号规则 | 2268 开头 |
| 示例账号 | 226801000000000101099 |
| 用途 | 商户资金账户，接收清算款项 |
| 开通方式 | 商户入驻成功后自动开通 |
| 内部结构 | 包含两个子户：可用解冻（BALANCE_AVAILABLE）和可用冻结（BALANCE_FREEZE） |

**商户子户余额公式：**

```
总余额 = 冻结子户 + 可用子户（解冻部分 + 冻结部分）
可提现金额 = 可用子户余额 - 业务预留金额
```

### 4. 商户保证金户（9903 开头）

| 属性 | 说明 |
|------|------|
| 账号规则 | 9903 开头 |
| 接口字段名 | TRADE_DEPOSIT_AVAILABLE |
| 用途 | 商户保证金账户，用于充值保证金 |
| 开通方式 | **方式一（推荐兜底）**：首次充值时静默开通；**方式二（推荐新商户）**：主动调用开户接口 |

**主动开户接口：** `ant.mybank.bkcloudfunds.account.open`

| 入参字段 | 说明 | 示例 |
|---------|------|------|
| MerchantId | 商户 ID | 网商商户号（2268 开头） |
| AcctType | 开户场景 | TRADE_DEPOSIT（保证金户） |
| OutTradeNo | 外部流水号 | {年月日}{自增ID}，如 20260605001 |

| 出参字段 | 说明 |
|---------|------|
| BankCardNo | 9903 开头的保证金户账号 |
| BranchNo | 开户行行号 |
| BranchName | 开户行名称 |
| BankCertName | 商户开户名称 |

也可通过商户子户查询接口（`ant.mybank.bkcloudfunds.account.query`）查询保证金户账号。开通后可在法人支付宝 → 网商银行小程序中查到。

## 操作步骤

### 查询平台清算专户余额

调用子户外标余额查询接口（3.59）：

| 入参字段 | 说明 | 示例 |
|---------|------|------|
| IsvOrgId | 平台 ID | 202211000000000004381 |
| BankCardNo | 平台 9903 开头的清算户卡号 | 99030360620900482009 |
| AccountType | 账户类型 | 01（子户卡号） |

### 查询商户子户余额

调用商户子户余额查询接口（3.9）：`ant.mybank.bkcloudfunds.subaccount.balance.query`

### 查询商户场景余额

调用账户场景余额查询接口：`ant.mybank.bkcloudfunds.merchant.scene.balance.query`，可查询可支付/可提现金额。

### 查询保证金充值明细

调用充值保证金查询接口：`ant.mybank.bkcloudfunds.vostro.charge.batchquery`

## 业务规则

### 资金流向关系

```
华夏银行备付金账户（POS 收银资金临时存放）
        │ 品牌方每日手动汇款
        ↓
平台清算专户（9903）
        │ 清算批次划拨
        ↓
商户子户（2268）
   ├──→ 协议代扣（体系内子户划转）→ 平台保证金户/结算户（8888）
   ├──→ 授权代付（体系外付款）→ 外部银行卡/支付宝
   └──→ 提现 → 商户绑定银行卡
```

### 限额规则

| 场景 | 限额 | 解决方案 |
|------|------|---------|
| 子户间划转 | 无上限 | - |
| 子户对外月度转账 | 约 40 万 | 同一营业执照注册多个商户号 |
| 授权代付各子场景 | 5/10/40/50/200 个收款方，总计 ≤200 | 按子场景独立绑定上限 |

### 商户子户出金额度

商户子户对外月度转账上限约为 **40 万元**。如需突破此限制，可在同一营业执照下注册多个商户号，实现额度扩容。

## 注意事项

1. **账户前缀不可混淆**：平台清算专户和商户保证金户都是 9903 开头，但属于不同层级的账户。平台清算专户是平台级账户，商户保证金户是商户级账户。
2. **商户保证金户开通时机**：推荐在商户首次充值时静默开通（兜底方案），新商户也可主动调用开户接口提前开通。
3. **余额明细获取方式**：商户子户余额明细**不能**通过来账明细查询接口获取，需通过资金账单下载申请接口（`ant.mybank.bkmbp.stmt.fund.stmt.apply`）获取，申请昨日及之前的账单，当日账单需次日申请。
4. **平台 8888 保证金户水位监控**：需设置余额预警线，避免因余额不足导致扣保垫付失败。
5. **提现卡 vs 结算户 vs 西进户**：三者是不同账户。提现卡是商户入驻时提交的银行卡；结算户是商户保证金户（9903 开头）；西进户是授权代付的过桥账户。

## 常见误区

1. **误区：9903 开头的账户都是同一个** — 实际上平台清算专户和商户保证金户都是 9903 开头，但它们是完全不同的账户，前者归平台所有，后者归各商户所有。
2. **误区：商户子户只有一个余额** — 商户子户内部包含可用解冻（BALANCE_AVAILABLE）和可用冻结（BALANCE_FREEZE）两部分，可提现金额需扣除业务预留金额。
3. **误区：子户对外转账没有限制** — 子户间划转无上限，但子户对外（如提现、授权代付）月度转账约 40 万上限。
4. **误区：来账明细查询接口可以查商户余额明细** — 来账明细查询接口（3.53）仅能查询平台清算专户（9903 开头）的入金明细，商户余额明细需通过资金账单下载申请接口获取。
5. **误区：商户入驻后保证金户自动开通** — 商户子户（2268）入驻后自动开通，但商户保证金户（9903）需要首次充值静默开通或主动调用开户接口。
