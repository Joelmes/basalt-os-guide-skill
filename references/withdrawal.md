# 提现

> 所属系统：Basalt OS 清算分账系统
> 关联接口：3.38 / 3.39 / 3.40 / 3.41 / 品牌商费用对账文件申请 / 协议授权代扣
> 最后更新：2026-06-08

---

## 概述

提现是商户将可用子户余额转出至其绑定银行卡的操作。在 Basalt OS 中，提现采用两步流程（申请 + 确认），网商银行收取 0.05%（万五）手续费，采用 D+1/D+2 后置收取方式——商户提现时全额到账，手续费由平台先行垫付，次日通过协议代扣从商户余额中追缴。

提现方案涉及的 4 个核心接口均为必接接口。

---

## 操作步骤

### 提现两步流程

#### 第一步：提现申请（平台 → 网商）

- **接口**：3.38 商户单笔提现申请接口（`ant.mybank.bkcloudfunds.withdraw.apply`）
- **功能**：发起一笔提现请求，调用后网商会自动给商户入驻时登记的负责人手机号发送短信验证码
- **核心入参**：

| 字段 | 说明 | 必填 | 示例 |
|------|------|------|------|
| IsvOrgId | 合作方机构号 | M | 网商分配 |
| MerchantId | 提现商户号（2268 开头） | M | - |
| OutTradeNo | 外部交易流水号（幂等键，同一流水号视为同一笔交易） | ME | - |
| TotalAmount | 提现总金额（单位：分） | M | - |
| Currency | 币种 | M | CNY |
| PlatformFee | 平台（品牌方）向商户收取的平台服务费，商户实收 = TotalAmount - PlatformFee | M | - |
| FeeCurrency | 币种 | M | CNY |
| Memo | 备注 | M | - |

- **核心出参**：

| 字段 | 说明 |
|------|------|
| OutTradeNo | 外部交易流水号 |
| OrderNo | 网商提现订单号（后续确认和查询用） |

- **重要**：此步骤不会扣减商户余额，仅触发短信验证。

#### 第二步：提现确认（平台 → 网商）

- **接口**：3.39 商户单笔提现确认接口（`ant.mybank.bkcloudfunds.withdraw.applyconfirm`）
- **前提**：已完成提现申请并获取 OrderNo
- **核心入参**：

| 字段 | 说明 | 必填 | 示例 |
|------|------|------|------|
| IsvOrgId | 合作方机构号 | M | - |
| MerchantId | 提现商户号 | M | - |
| OutTradeNo | 外部交易号 | ME | - |
| OrderNo | 网商提现订单号（申请接口返回） | M | - |
| PlatformFee | 平台服务费 | M | - |
| FeeCurrency | 币种 | M | CNY |
| TotalAmount | 订单金额（单位：分） | M | - |
| Currency | 币种 | M | CNY |
| SmsCode | 短信动态码 | M | 888888（测试环境） |
| Memo | 备注 | M | - |

- **核心出参**：

| 字段 | 说明 |
|------|------|
| OutTradeNo | 外部交易流水号 |
| OrderNo | 网商提现订单号 |
| TotalAmount | 订单金额（分） |
| Currency | 币种 |

- **重要**：确认接口调用成功后，商户子户余额立即扣减，网商开始向商户清算卡转账。

#### 获取提现结果

**方式 1：被动通知**

- **接口**：3.40 提现结果通知接口（`ant.mybank.bkcloudfunds.withdraw.notify`）
- 网商异步通知提现最终结果（SUCCESS / FAIL）
- 通知参数包含：MerchantId、OrderNo、TotalAmount、Status、BankCardNo、WithdrawApplyDate、WithdrawFinishDate 等
- 通知失败会重试

**方式 2：主动查询**

- **接口**：3.41 单笔提现查询接口（`ant.mybank.bkcloudfunds.withdraw.query`）
- 入参：IsvOrgId + MerchantId + OutTradeNo（或 OrderNo）
- 出参 Status 枚举：

| 状态 | 说明 |
|------|------|
| SUCCESS | 提现成功 |
| FAIL | 提现失败 |
| DEALING | 处理中，需继续查询 |

### 完整流程时序

```
平台端                  网商云资金                商户端
  |                         |                         |
  |--1.提现申请接口-->       |                         |
  |   (发起提现申请)          |                         |
  |                         |--> 发送短信验证码       |
  |                         |                         |
  |<-- 返回订单号 --         |                         |
  |                         |                         |
  |--2.提现确认接口-->       |                         |
  |   (输入短信验证码)         |                         |
  |                         |                         |
  |                         |-- 处理提现              |
  |                         |                         |
  |<--3. 提现结果通知--       |                         |
  |   (异步通知)              |                         |
  |                         |                         |
  |--4.提现结果查询-->       |                         |
  |   (主动查询)              |                         |
  |                         |                         |
```

## 业务规则

### 提现状态

提现查询接口返回的 Status 有三种终态/中间态：

| 状态 | 含义 | 后续操作 |
|------|------|---------|
| SUCCESS | 提现成功 | 记录成功，流程结束 |
| FAIL | 提现失败 | 记录失败原因（ErrorDesc），可重新发起 |
| DEALING | 处理中 | 继续轮询查询直至终态 |

提现通知接口（3.40）只返回 SUCCESS 或 FAIL 两种终态。

### 费率与手续费收取机制

#### 费率

| 收费项目 | 费率 | 收取方式 | 承担方 |
|---------|------|---------|--------|
| **提现手续费** | 0.05%（万五） | D+1 从平台 8888 保证金户（结算户）自动扣除 | 商户（平台先行垫付，D+2 协议代扣向商户追缴） |
| 授权代付手续费 | 0.05%（万五） | D+1 从平台 8888 保证金户（结算户）自动扣除 | 平台（不建议转嫁） |
| 协议代扣 | 0.00%（免费） | - | - |

#### 手续费收取全流程（D+1 / D+2）

```
D 日：商户发起提现
    - 提现金额全额到商户提现卡（银行卡）
    - 商户不感知手续费

D+1 日：网商扣收手续费
    - 网商从平台 8888 保证金户（结算户）收取前一日所有商户产生的提现手续费
    - 平台先行垫付

D+2 日凌晨 3 点：品牌方拉取对账文件并追缴
    - 调用品牌商费用对账文件申请接口（ant.mybank.bkmbp.stmt.fee.file.apply）
    - 拉取前一日品牌方向网商银行已支付的手续费账单
    - 从账单中筛选"提现（小程序模式）"和"提现（API 模式）"手续费记录
    - 与本地提现通知记录比对，找出新增订单
    - 通过单笔提现查询接口获取提现商户信息
    - 通过协议授权代扣接口向商户逐笔收取手续费
```

#### 协议代扣收取手续费的核心入参

| 参数 | 值 | 说明 |
|------|-----|------|
| PayerId | 提现商户的网商商户号（2268 开头） | 付款方 |
| PayerType | MERCHANT | 付款方类型 |
| PayeeId | 平台 8888 开头的保证金户（结算户） | 收款方 |
| PayeeType | PLATFORM | 收款方类型 |

#### 品牌商费用对账文件注意事项

- 对账文件**只包含平台已支付的手续费账单**，未支付的账单不会展示
- 需从账单中区分"提现（小程序模式）"和"提现（API 模式）"两种手续费类型
- 协议代扣收取的手续费记录会展示在平台的付款记录中，品牌方可通过付款记录查看每笔代扣明细

### 商户全额到账

- 商户提现时，提现金额**全额到达**商户提现卡（银行卡）
- 网商银行不直接从提现金额中扣除手续费
- 手续费通过 D+1/D+2 后置机制另行收取
- **PlatformFee** 字段为平台（品牌方）自行向商户收取的服务费，与网商手续费无关；商户实收 = TotalAmount - PlatformFee

### 银行卡绑定

- 商户提现卡为商户入驻时提交的银行卡账户
- 银行卡信息通过商户信息修改接口（`ant.mybank.merchantprod.merch.updateMerchant`）可修改
- 提现资金最终到达的银行卡即为入驻时绑定的清算卡
- 修改银行卡信息后提现到新卡

### 短信验证

- 提现申请接口调用后，网商自动给商户入驻时登记的负责人手机号发送短信验证码
- 提现确认接口需传入正确的短信验证码
- **测试环境验证码**：`888888`
- 免短验后可不再上送 SmsCode

### 提现前提条件

1. 已完成商户入驻并获取网商商户号（MerchantId）
2. 通过余额查询接口确认可用子户金额 > 0
3. 商户已完成协议代扣签约（用于 D+2 手续费追缴）

### 金额限制

- **最大单笔提现金额**：1000 万
- **可提现金额** = 可用子户余额 - 业务预留金额

## 注意事项

### 幂等控制

- 提现申请：OutTradeNo 同一交易号被视为同一笔交易
- 提现确认：OrderNo 同一订单号可重复确认

### 余额扣减时机

- 提现申请阶段**不会扣减余额**
- 确认接口调用成功后才扣款
- 如确认接口提示成功后，立即扣减商户子户余额

### 通知处理

- 提现通知本身**不含手续费字段**，手续费信息必须通过 D+2 对账文件获取
- 收到通知后建议同步调用查询接口确认
- 通知失败会按重试策略重试：1, 3, 5, 10, 15, 30, 60, 300, 600, 900, 1800, 3600（单位：秒）

### 协议代扣失败处理

- 如果 D+2 协议代扣收取手续费失败（商户余额不足），需人工介入
- 平台已垫付的手续费存在回收风险

### 商户自助提现

- 商户也可在网商小程序中自助提现
- 无论通过 API 还是小程序提现，接口都必须接入
- 小程序提现同样会产生手续费，走相同的 D+1/D+2 收取流程

## 常见误区

### 误区 1：提现时直接从提现金额中扣手续费

**错误**：认为商户提现到账金额 = 提现金额 - 手续费。

**正确**：商户提现时全额到账，手续费由平台 D+1 从 8888 保证金户先行垫付，D+2 再通过协议代扣从商户余额追缴。商户在提现操作时完全不感知手续费。

### 误区 2：提现申请阶段就会扣减余额

**错误**：调用提现申请接口后就认为余额已扣减。

**正确**：提现申请接口不会扣减余额，只有确认接口调用成功后才会扣款。

### 误区 3：提现通知中包含手续费信息

**错误**：期望从提现结果通知接口获取手续费金额。

**正确**：提现通知不含手续费字段。手续费信息必须通过 D+2 品牌商费用对账文件（`ant.mybank.bkmbp.stmt.fee.file.apply`）获取。

### 误区 4：提现手续费由平台承担

**错误**：认为平台 8888 户被扣的手续费就是平台的成本。

**正确**：提现手续费最终由商户承担。平台只是先行垫付（D+1），次日（D+2）通过协议代扣向商户追缴。授权代付的手续费才建议由平台承担（不建议转嫁给商户）。

### 误区 5：测试环境需要真实短信验证码

**错误**：测试时等待真实短信验证码。

**正确**：测试环境统一验证码为 `888888`，直接使用即可。

### 误区 6：对账文件包含未支付的手续费

**错误**：期望 D+2 对账文件中包含所有产生的手续费记录。

**正确**：品牌商费用对账文件**只包含平台已支付的手续费账单**，未支付的账单不会出现在文件中。
