协议代扣和授权代付有什么区别？
协议代扣是体系内子户间划转（免费），授权代付是向体系外付款（0.05%）。自动路由至 payments.md 返回完整对比。
清算文件命名规则是什么？
CARE{日期}_{序号}，如 CARE20260509_01。每日上传限制 50 次。路由至 clearing.md 获取完整规则。
deposit.notify 类型 1 和类型 5 有什么区别？
类型 1 是余额不足扣保通知（外部请求单号为空），类型 5 是退保结果通知（有外部请求单号）。路由至 clearing.md。
提现手续费怎么收？商户看到的是全额吗？
网商收万五，D+1 从 8888 户扣，D+2 通过协议代扣从商户追缴。商户看到全额到账。路由至 merchants.md + global-rules.md。

GET STARTED

一行命令，所有 Agent 通用

三种安装方式，适配所有 Agent。有终端用方式一，能上网用方式二，直接装文件用方式三。

⚡ 方式一：Shell 安装（推荐）

适用于有终端能力的 Agent（Claude Code、Gemini CLI、Cursor、Codex、Cline 等）。

npx skills add Joelmes/basalt-os-guide-skill -g 点击复制

需要 Node.js 环境，-g 表示全局安装。

🔗 方式二：URL 安装（通用）

适用于所有能访问网页的 Agent。复制下面的指令发给你的 Agent，它会自动完成安装。

安装这个 skill https://github.com/Joelmes/basalt-os-guide-skill，包括所有子文件 点击复制

只要 Agent 能访问网页就能用。Agent 会自动读取 GitHub 仓库内容并创建本地文件。

📦 方式三：Release 下载（离线安装）

从 GitHub Release 下载最新版 .skill 文件，直接拖入 Agent 窗口即可安装。适合网络受限或离线环境。

$ user: 协议代扣和授权代付有什么区别？
# 自动路由: payments.md
agent: 两种付款方式的核心区别：
1. 协议代扣（系统术语：余额划转）
- 体系内子户间划转，免费
- 用于服务费、采购分配等场景
- 商户需主动签约
2. 授权代付（系统术语：授权代付）
- 向体系外付款，手续费 0.05%
- 用于房租、工资、报销等外部支付
- 平台发起签约
